### LAB5 PROJECT: AVATAR NA'VI HABITAT SUITABILITY MODEL (THAILAND)

ส่วนที่ 1: Problem Statement (ความสำคัญและที่มาของปัญหา)
เมื่อแพนโดร่าล่มสลาย: แผนที่การอยู่รอดของชาวนาวีในประเทศไทย
โปรเจกต์นี้เป็นการจำลองสถานการณ์ว่า หากชาวนาวีต้องอพยพจากแพนโดร่า และต้องมาอาศัยอยู่บนโลก
โดยเฉพาะในประเทศไทย
พื้นที่ใดจะเหมาะสมกับการดำรงชีวิตของพวกเขา

ประเทศไทยมีความหลากหลายของระบบนิเวศ เช่น ป่าฝน ภูเขา และชายฝั่งทะเล ซึ่งสามารถวิเคราะห์ได้ผ่านข้อมูลดาวเทียม เช่น NDVI ปริมาณฝน อุณหภูมิ และความสูง

โครงการนี้จึงนำเสนอการสร้างแบบจำลองเพื่อค้นหาและประเมินพื้นที่ตั้งถิ่นฐานที่เหมาะสม สำหรับประชากร 3 กลุ่มที่มีเงื่อนไขทางนิเวศวิทยาและลักษณะทางภูมิประเทศที่ต้องการแตกต่างกัน อ้างอิงสมมติฐานทางกายภาพจากภาพยนตร์ Avatar ได้แก่ Omaticaya (เผ่าป่า) Ash Tribe (เผ่าภูเขา) และ Metkayina (เผ่าทะเล)

โครงสร้างของแบบจำลองนี้สามารถนำไปเป็นรากฐานในการประยุกต์ใช้จริง สำหรับการจัดการพื้นที่อนุรักษ์ การหาพื้นที่อพยพ หรือการจัดสรรที่ดินทำกินในสถานการณ์วิกฤตทางสิ่งแวดล้อมได้

ส่วนที่ 2: Research Questions (คำถามการวิจัย)
โครงการนี้มุ่งเน้นตอบคำถามเชิงพื้นที่ (Spatial Analysis) ดังต่อไปนี้:
Where: พื้นที่บริเวณใดในประเทศไทยที่มีสภาพแวดล้อมทางกายภาพเหมาะสมที่สุด ต่อการตั้งถิ่นฐานของประชากรทั้ง 3 กลุ่มหลัก โดยอิงตามเกณฑ์ทางนิเวศวิทยาเฉพาะกลุ่ม ?
How much: พื้นที่ที่มีระดับความเหมาะสมสูง (Suitability Score > 0.5 - 0.6) มีขนาดของผืนพื้นที่ที่เชื่อมต่อกัน เพียงพอที่จะรองรับการตั้งถิ่นฐานของประชากรขั้นต่ำจำนวน 300 คนหรือไม่ ?

ส่วนที่ 4: Methodology Flow
 การเตรียมข้อมูล: นำเข้าข้อมูลภาพถ่ายดาวเทียม กรองช่วงเวลา (ปี 2020-2023) ตัดขอบเขตด้วย Shapefile ประเทศไทย และดำเนินการปรับมาตรฐานข้อมูล (Min-Max Normalization) ให้อยู่ในช่วงคะแนน 0 ถึง 1
 การวิเคราะห์ข้อมูลหลัก: ประยุกต์ใช้วิธีการตัดสินใจแบบหลายเกณฑ์ โดยกำหนดค่าน้ำหนัก ของแต่ละตัวแปรให้สอดคล้องกับพฤติกรรมประชากรเป้าหมาย พร้อมทั้งใช้ข้อมูลพิกัดน้ำทะเลและ Land Cover ในการสร้างเงื่อนไขจำกัดขจัดพื้นที่ที่ไม่เกี่ยวข้องออก
 การวิเคราะห์ขีดความสามารถรองรับ: คัดกรองพื้นที่ด้วยคำสั่ง connectedPixelCount เพื่อตรวจสอบขนาดผืนพื้นที่ที่เชื่อมต่อกัน (Patch Size) โดยคัดพื้นที่ขนาดเล็กกว่าเกณฑ์ขั้นต่ำ 200-400 ตร.กม. ออกจากแบบจำลอง
 การส่งออกผลลัพธ์: ส่งออกผลลัพธ์เชิงพื้นที่รูปแบบ Raster (GeoTIFF) ไปยัง Google Drive เพื่อนำไปวิเคราะห์และจัดทำสัญลักษณ์ (Symbology) ขั้นสุดท้ายในซอฟต์แวร์ ArcGIS Pro

ส่วนที่ 5: การประเมินความเป็นไปได้และความเสี่ยง
 ความเสี่ยงหลัก: ขีดจำกัดด้านหน่วยความจำของเซิร์ฟเวอร์ GEE ระหว่างการประมวลผลขนาดพื้นที่ และการส่งออกข้อมูลระดับประเทศ
Plan B: หากเกิดข้อจำกัดในการประมวลผล Vector จะปรับไปใช้การส่งออกผลลัพธ์ในรูปแบบ Raster (GeoTIFF) เพียงอย่างเดียว เนื่องจากใช้ทรัพยากรน้อยกว่า คงความถูกต้องของค่าคะแนน (0.0-1.0) ได้ดีกว่า และสามารถนำไปปรับแต่งใน ArcGIS Pro ได้อย่างมีประสิทธิภาพ

1. ถ้าต้องลด Scope ของโครงการลง 50% จะตัดอะไรออกก่อน และ Core ที่ต้องรักษาไว้คืออะไร?
 ส่วนที่จะตัดออก: จะทำการลดขอบเขตโดยตัด การวิเคราะห์ขีดความสามารถในการรองรับของพื้นที่ รวมถึงตัดกระบวนการ Export ข้อมูลไปทำต่อใน ArcGIS Pro ออก
Core ที่ต้องรักษาไว้: แกนหลักที่จะต้องดำเนินการให้แล้วเสร็จคือ อัลกอริทึมการซ้อนทับข้อมูลและวิเคราะห์แบบหลายเกณฑ์ (MCDA) บน Google Earth Engine เพื่อให้บรรลุวัตถุประสงค์ในการสร้างแผนที่ฐานที่สามารถแยกแยะความเหมาะสมของพื้นที่สำหรับประชากรทั้ง 3 กลุ่มได้อย่างชัดเจน
2. มีโครงการหรือ Paper ใดที่ทำสิ่งคล้ายกันแล้ว? ผลลัพธ์ของโครงการนี้จะต่างหรือเพิ่มเติมจากนั้นอย่างไร?
ความคล้ายคลึง: โครงการนี้มีพื้นฐานใกล้เคียงกับงานวิจัยด้าน "การสร้างแบบจำลองการกระจายชนิดพันธุ์ (Species Distribution Modeling: SDM)" หรือการประเมินแหล่งที่อยู่อาศัยของสัตว์ป่าสงวน
ความแตกต่างและส่วนเพิ่มติม: โครงการนี้มีความแตกต่างอย่างมีนัยสำคัญในแง่ของ การบูรณาการสมมติฐานเชิงพฤติกรรม (Behavioral Assumptions) เข้ามาเป็นพารามิเตอร์หลัก เช่น การปรับโมเดลความลาดชัน (Slope) ให้เก็บคะแนนง่ายขึ้นสำหรับกลุ่มที่ชอบความโดดเดี่ยว หรือการกำหนดรัศมีห่างชายฝั่ง โครงการนี้จึงโดดเด่นในการแสดงศักยภาพการทำ Feature Engineering ทางภูมิศาสตร์ ที่สามารถดัดแปลงให้ยืดหยุ่นตามโจทย์และบริบทจำลองที่ซับซ้อนได้อย่างเป็นรูปธรรม
3. ใครคือ End-user จริงๆ ของผลลัพธ์? พวกเขาจะใช้แผนที่/โมเดลที่ได้ไปทำอะไรจริงๆ?

มิติอุตสาหกรรมสร้างสรรค์: ทีมนักพัฒนาเกม ผู้กำกับศิลป์ หรือนักเขียน ที่ต้องการข้อมูลอ้างอิงเชิงภูมิศาสตร์ของสภาพแวดล้อมจริงในประเทศไทย เพื่อออกแบบฉากหรือโลกเสมือน (World-building) ให้มีความสมจริง
 มิติการประยุกต์ใช้จริงในสังคม (Real-world Impact): นักผังเมือง เจ้าหน้าที่กรมป่าไม้ หรือหน่วยงานบรรเทาสาธารณภัย สามารถนำโครงสร้างอัลกอริทึมของโครงการนี้ไปเปลี่ยนตัวแปร เพื่อใช้ค้นหาพื้นที่รองรับผู้ลี้ภัยทางสภาพภูมิอากาศ การจัดสรรที่ดินทำกินใหม่ให้กลุ่มชาติพันธุ์ หรือกำหนดแนวเขตป่ากันชนในสภาวะวิกฤตได้อย่างมีประสิทธิภาพ

 Proposal 3 ข้อ (Core Concept ของโครงการ)
1. ปัญหาคืออะไร และทำไมมันสำคัญ?
ตอบ: ปัญหาคือ ประเทศไทยยังขาดแบบจำลองเชิงพื้นที่ ที่สามารถค้นหาและประเมินพื้นที่ตั้งถิ่นฐาน สำหรับกลุ่มประชากรหรือสิ่งมีชีวิตที่มีเงื่อนไขทางนิเวศวิทยาและภูมิประเทศที่เฉพาะเจาะจงอย่างสุดขั้ว ความสำคัญของโครงการนี้คือ อัลกอริทึมและตรรกะที่พัฒนาขึ้น จะทำหน้าที่เป็น โมเดลต้นแบบ ที่สามารถนำไปปรับพารามิเตอร์เพื่อแก้ปัญหาในโลกจริงได้ เช่น การหาพื้นที่อพยพให้ผู้ลี้ภัยทางสภาพภูมิอากาศ หรือการจัดสรรพื้นที่อนุรักษ์แนวกันชน

2. จะใช้ข้อมูลอะไรและวิธีการใด เพื่อตอบคำถามอะไร?
ตอบ: โครงการนี้จะใช้ข้อมูลภาพถ่ายดาวเทียมและข้อมูลทางภูมิศาสตร์บน Google Earth Engine (ปี 2020-2023) ได้แก่ ข้อมูลความสูง/ความชัน ดัชนีพืชพรรณ ปริมาณฝน และประเภทการใช้ที่ดิน โดยประยุกต์ใช้วิธี การวิเคราะห์แบบหลายเกณฑ์ (MCDA - Multi-Criteria Decision Analysis) ผสานกับการคำนวณ ขีดความสามารถรองรับของพื้นที่ เพื่อตอบคำถามว่า พื้นที่บริเวณใดในประเทศไทยที่มีขนาดใหญ่เพียงพอ และมีสภาพแวดล้อมเหมาะสมที่สุดต่อการดำรงชีวิตของประชากรแต่ละกลุ่มเป้าหมาย?

3. ผลลัพธ์จะมีหน้าตาอย่างไร และใครได้ประโยชน์?
ตอบ: ผลลัพธ์ที่ได้คือ แผนที่แสดงระดับความเหมาะสมของพื้นที่ ในรูปแบบข้อมูล Raster (GeoTIFF) ที่ผ่านการกรองข้อมูลพิกเซลให้เหลือเฉพาะพื้นที่ที่ผ่านเกณฑ์ประชากรขั้นต่ำ โดยสามารถนำไปไล่เฉดสีเพื่อวิเคราะห์เชิงลึกต่อใน ArcGIS Pro ได้ ผู้ที่ได้รับประโยชน์ ได้แก่ นักผังเมือง หน่วยงานจัดการภัยพิบัติ หรือนักนิเวศวิทยา ที่สามารถนำ Data Pipeline นี้ไปปรับใช้กับงานวิจัยของตนเอง ไปจนถึงนักสร้างสรรค์และนักออกแบบสื่อ  ที่ต้องการอ้างอิงสภาพแวดล้อมจากข้อมูลภูมิศาสตร์จริง
